# Веб-приложение для учета сотрудников в организациях

## 1. Инструментарий
Для реализации проекта выбран следующий стек и окружение:

- Операционная система: Windows 10.
- Среда разработки (IDE): Visual Studio Code.
- База данных: PostgreSQL 17.
- Управление контейнерами: Docker Desktop для Windows.
- Система контроля версий: Git.
- Дизайн БД: Draw.io (файл схемы находится в /docs/schema.drawio).
- Backend: Node.js v22.
- Frontend: Vue.js 3.5 + Quasar + Vite.

## 2. Схема базы данных
- Схема базы данных находится по адресу docs/schema.drawio

## 3. Работа с Git
В ходе работы над проектом используются следующие основные команды системы контроля версий:

- git init – инициализация нового репозитория.
- git clone <url> – клонирование удаленного репозитория.
- git status – проверка состояния файлов (измененные, добавленные, удаленные).
- git add <file> – добавление конкретного файла в индекс (staging area).
- git add . – добавление всех измененных файлов в индекс.
- git commit -m "message" – фиксация изменений с сообщением.
- git push – отправка изменений в удаленный репозиторий (GitHub).
- git pull – получение изменений из удаленного репозитория.

Работа с ветками:
- git branch – просмотр списка веток.
- git branch <name> – создание новой ветки.
- git checkout <branch> – переключение на ветку.
- git checkout -b <name> – создание и переключение на новую ветку.
- git merge <branch> – слияние указанной ветки в текущую.

История и откат:
- git log – просмотр истории коммитов.
- git log --oneline – краткий просмотр истории.
- git diff – просмотр различий между изменениями._

## 4. Структура проекта
- /api - Backend (NestJS)
- /app - Frontend (Vue.js)
- /containers - Dockerfile для сборки образов
- /docs - Документация и схемы
- /migrations - SQL миграции базы данных
- docker-compose.yml - Конфигурация запуска сервисов
